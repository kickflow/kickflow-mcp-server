ultrathink : Renovate の PR をチェックして main ブランチにマージして問題ないか確認して : $ARGUMENTS

## 手順

1. PR番号を解析
   - $ARGUMENTS から PR番号を抽出
   - PR番号が指定されていない場合はエラー

2. GitHub CLI を使用して PR情報を取得
   - `gh pr view <PR番号> --json` で詳細情報を取得
   - タイトル、説明、変更ファイルのリストを確認
   - Renovate によるPRかどうかを確認

3. CI ステータスの確認
   - `gh pr checks <PR番号>` で CI の実行状況を確認
   - 失敗しているワークフローがないかチェック
   - 各ワークフローの結果をリスト表示

4. 依存関係の変更内容を分析
   - `gh pr diff <PR番号>` で差分を取得
   - package.json, package-lock.json の変更を解析
   - メジャーバージョンアップの検出
   - セマンティックバージョニングに基づく破壊的変更の可能性を評価

5. 更新されるパッケージの詳細情報を収集
   - 各パッケージのバージョン変更を表示
   - 可能な場合、リリースノートやCHANGELOGへのリンクを提供
   - BREAKING CHANGES の有無を確認

6. レポート生成
   - 安全にマージできるかの総合判定
   - 注意が必要な変更のハイライト
   - 推奨アクション（すぐマージ可能、レビューが必要、など）

## オプション

- `--skip-ci-check`: CI確認をスキップ
- `--verbose`: 詳細な出力を表示

## その他ルール

- **IMPORTANT** : 日本語でやりとりすること
- **IMPORTANT** : PR番号が指定されていない場合は適切にエラーメッセージを表示
- **IMPORTANT** : Renovate以外のPRの場合は警告を表示
- **IMPORTANT** : メジャーバージョンアップがある場合は特に注意喚起
- **IMPORTANT** : CI が失敗している場合は、マージ不可と判定
