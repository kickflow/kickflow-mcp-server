現在のブランチをGitHubにプッシュし、プルリクエストを作成します。
ユーザーに質問がある場合は、1つずつ必ず尋ねてください。
以下の手順を必ず遵守してください。勝手に手順を飛ばしたりコミットすることは禁止です。

## 手順

1. 現在のブランチを確認し、GitHubにプッシュします。
   - `git branch --show-current` で現在の[ブランチ名]を確認します。
   - `git push origin [ブランチ名]` を実行します。

2. Pull Requestの[マージ先ブランチ]をユーザーに尋ねてください。その際、以下のコマンドでRemoteのbranchをすべて表示して「どのブランチにマージしますか？」と質問してください。
   - `git fetch origin && git branch --remotes | grep origin | grep -v -- '->' | grep -v 'renovate/' | sed 's| *origin/||'`

3. 現在のブランチで発生した差分を以下のコマンドで確認して、実装内容を把握して[やったこと]として簡潔に要約してください。
   - `git diff origin/[マージ先ブランチ]...[ブランチ名]`

4. 次に[PRの説明]を[プロジェクトフォルダ]/.github/pull_request_template.mdのテンプレートに従って作成します。
   - テンプレートにあるコメントに従って、必要な情報を記入してください。
   - テンプレート内の[やったこと]は、3.で確認した内容をコメントの指示に従って記入してください

5. Pull Requestを作成します。
   - `gh pr create --base [マージ先ブランチ] --title "[PRのタイトル]" --body "[PRの説明]"` を実行します。
   - [PRのタイトル]は、実装内容を簡潔に表現した日本語のタイトルを作成してください。
   - [PRの説明]は、4.で作成した内容をそのまま貼り付けてください。
